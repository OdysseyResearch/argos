# Implementation Plan: Argos v0.1 MCP Security Proxy MVP

**Branch**: `001-mcp-proxy-mvp` | **Date**: 2026-04-24 | **Spec**: `specs/001-mcp-proxy-mvp/spec.md`

---

## Summary

A single Rust binary (`argos-proxy`) that sits transparently between any MCP client and any MCP
server, intercepting tool calls and resource accesses, evaluating them against a TOML policy
(deny by default), and writing every decision to an append-only Merkle-chained JSONL audit log.
Supports stdio mode (wraps a local MCP server subprocess) and HTTP/SSE mode (reverse proxy in
front of a remote server). Exposes the policy engine and audit writer as a public library crate.

## Architecture: Request Interception Pipeline

Every intercepted request flows through the same pipeline regardless of transport mode:

```
Transport adapter (stdio | http)
        │ McpFrame (raw bytes)
        ▼
proxy::intercept()
        │
        ├─ parse method field only
        │
        ├─ if pass-through method (initialize, ping, tools/list, etc.)
        │       └─ forward raw bytes unchanged → transport adapter
        │
        └─ if intercepted method (tools/call, resources/read, resources/list, resources/subscribe)
                │ parse full params
                │
                ├─ policy::engine::evaluate()   [no lock, concurrent]
                │
                ├─ audit::writer::write()        [Mutex held for hash+write only]
                │
                └─ if Allow or Redact → forward (redacted) request → upstream
                   if Block → return JSON-RPC error to client
```

**Concurrency model**: Each request is handled in a separate Tokio task. Policy evaluation is
`&self` (no mutation, no lock). The audit Mutex is held for microseconds. In-flight tasks share
`Arc<PolicyEngine>` and `Arc<AuditWriter>`.

## Key Implementation Notes

### Content-Length framing (stdio)

Custom `tokio_util::codec::Decoder` implementation:

- State machine: `ReadingHeader` → `ReadingBody(n)`
- Header parsing: read bytes until `\r\n\r\n`, extract `Content-Length: N`
- Body parsing: consume exactly N bytes as UTF-8 JSON

### Graceful shutdown

```rust
tokio::select! {
    _ = tokio::signal::ctrl_c() => {},
    _ = sigterm.recv() => {},           // SIGTERM on Unix
}
cancellation_token.cancel();
join_all(task_handles).await;           // drain in-flight requests
audit_writer.flush().await?;            // flush OS buffer to disk
process::exit(0);
```

### `entry_hash` convention

1. Serialize entry with `entry_hash: ""` (empty string)
2. SHA-256 of UTF-8 bytes → `"sha256:<hex>"`
3. Set field, re-serialize, write as single JSONL line
4. Documented in `contracts/audit-log.md`; tested in `tests/audit_chain.rs`

### Argument truncation

```rust
let serialized = serde_json::to_string(&arguments)?;
if serialized.len() > config.max_arg_bytes {
    entry.arguments = serde_json::json!("<truncated>");
    entry.arguments_truncated = true;
} else {
    entry.arguments = arguments;
    entry.arguments_truncated = false;
}
```

### HTTP mode upstream forwarding

For SSE streams: `reqwest::Response::bytes_stream()` piped into an `axum` streaming response
body. The proxy evaluates the *request* (tool call or resource access) before forwarding —
response bytes are passed through unmodified without evaluation.

| #  | Constraint                                       | How satisfied                                                                                      |
| -- | ------------------------------------------------ | -------------------------------------------------------------------------------------------------- |
| 1  | `org_id`, `tenant_id` in audit schema (nullable) | `AuditEntry` fields, always `None` in v0.1                                                         |
| 2  | Policy `version` field, validated at load        | `PolicyMeta.version`, hard error on unknown value                                                  |

| 4  | SHA-256 hash function                            | `sha2::Sha256` — not replaceable without format change                                             |
| 5  | OTel span emission architecturally possible      | `proxy::intercept()` is a natural span boundary; no instrumentation yet but pipeline is not closed |
| 6  | `argos` buildable as binary + library            | `[[bin]]` + `[lib]` in `Cargo.toml`                                                                |
| 7  | MCP error responses are spec-compliant JSON-RPC  | `{"jsonrpc":"2.0","id":...,"error":{"code":-32000,"message":"..."}}`                               |
| 8  | HTTP/SSE mode accepts TLS config                 | `--tls-cert` and `--tls-key` flags validated at startup                                            |
| 9  | `rotation_marker` entry type in schema           | `RotationMarkerEntry` struct defined; not emitted in v0.1                                          |
| 10 | Policy rules have `tags` field                   | Required field in `PolicyRule`; `[]` acceptable                                                    |
